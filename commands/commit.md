現在の変更差分を確認し、適切なコミットメッセージを生成してcommit & pushするコマンド。

## 手順

### 1. 変更差分の確認

以下のコマンドを並列実行して現状を把握する:
- `git status`（未追跡ファイル・変更ファイルの一覧）
- `git diff --stat`（変更の統計）
- `git log --oneline -5`（直近のコミットメッセージのスタイル確認）

変更がない場合は「コミットする変更がありません」と表示して終了。

### 2. 変更内容の分析

変更されたファイルの内容を分析し、以下を特定する:
- 変更の種類（新規作成、更新、削除、移動）
- 影響を受けたディレクトリ（gtd/, zettelkasten/ など）
- 変更の目的（inbox処理、ノート作成、リンク設定、レビューなど）

### 3. コミットメッセージの生成

以下のルールでコミットメッセージを生成する:
- 1行目: 英語で簡潔な要約（50文字以内推奨）
- 空行
- 本文: 変更内容の箇条書き（日本語の内容は英語で要約）
- 末尾に `Co-Authored-By: Claude Opus 4.6 <noreply@anthropic.com>`

### 4. ステージング

変更ファイルをすべてステージングする。ただし以下は除外:
- `.env` などの機密ファイル
- `.DS_Store` などのシステムファイル

### 5. コミット & プッシュ

- `git commit` を実行
- `git push` を実行
- 完了メッセージを表示:「コミット `<hash>` をプッシュしました」

## 注意事項
- 変更がない場合はコミットしない
- コミットメッセージは直近のコミットのスタイルに合わせる
- プッシュ先はカレントブランチのリモート追跡ブランチ
